### Раздел 4 «Информационные технологии»


#### Вопросы формирующего оценивания усвоения материала


1. **Чем отличается машинное обучение от традиционного программирования?**  
   * **Традиционное программирование:** человек пишет жёсткие правила («если X, то Y»), а программа их исполняет.  
   * **Машинное обучение:** человек даёт данные и цель, а алгоритм сам находит закономерности и «пишет» правила. Программа не жёстко запрограммирована, а «учится» на примерах.

2. **Обучение с учителем vs. обучение без учителя**  
   * **С учителем (supervised):** есть размеченные данные (каждому примеру — «правильный ответ»). Цель — предсказать ответ для новых примеров.  
     *Пример:* классификация писем на «спам»/«не спам».  
   * **Без учителя (unsupervised):** нет разметок. Цель — найти скрытые структуры (кластеры, ассоциации).  
     *Пример:* группировка покупателей по поведению без заранее заданных категорий.


3. **Что такое векторизация текста?**  
   Преобразование слов/предложений в числовые векторы (списки чисел) так, чтобы семантически близкие слова имели близкие векторы. Модель «понимает» смысл через геометрию векторов: расстояние между ними отражает смысловую близость.

4. **Какой тип ML для системы рекомендаций онлайн‑магазина?**  
   **Обучение без учителя (кластеризация)** или **обучение с учителем (классификация/регрессия)**.  
   * *Кластеризация:* группируем пользователей с похожим поведением и предлагаем товары, популярные в их кластере.  
   * *Классификация/регрессия:* предсказываем вероятность покупки конкретного товара для пользователя на основе его истории (размеченные данные: «пользователь X купил товар Y»).

5. **Процесс взаимодействия пользователя с языковой моделью через промпты**  
   1. Пользователь формулирует запрос (промпт).  
   2. Модель преобразует текст в векторы.  
   3. Модель обрабатывает последовательность векторов через слои нейросети.  
   4. Модель генерирует ответ (последовательно предсказывает следующие слова/токены).  
   5. Ответ декодируется в текст и показывается пользователю.

6. **Три ключевых вопроса для эффективного промпта**  
   * **Что?** — Чётко указать задачу (написать, объяснить, перевести, придумать и т.п.).  
   * **Почему?** — Обозначить цель/контекст (для кого, зачем, какой стиль).  
   * **Как?** — Задать формат, длину, структуру ответа (список, таблица, 3 предложения и т.п.).

7. **Почему две модели дают разные ответы на один вопрос?**  
   Из‑за различий в:  
   * **Обучающих данных** (разные корпуса, языки, темы).  
   * **Архитектуре и гиперпараметрах** (размер модели, количество слоёв).  
   * **Процессе обучения** (случайные инициализации, порядок данных).  
   Модель «помнит» то, что видела в данных, поэтому ответы зависят от её «опыта».


8. **Как улучшить промпт «Нарисуй космический корабль»?**  
   Добавить детали:  
   > «Нарисуй детализированный концепт-арт футуристического космического корабля в стиле научной фантастики, с яркими двигателями и металлическими панелями. Фон — звёздное небо и планета. Используй реалистичные цвета и тени.»

### Задания для текущего оценивания изученного материала

1. **Заполните пропуски:**  
   Машинное обучение — это подход к разработке информационных систем, в котором заранее подготовленную **модель** адаптируют к работе на основе **обучающих** данных.


2. **Соответствие видов ML и описаний:**  
   ```
   а | б | в  
   --|--|--
   2 | 1 | 3  
   ```  
   * а) Обучение с учителем → 2. Размеченные данные.  
   * б) Обучение без учителя → 1. Неразмеченные данные.  
   * в) Обучение с подкреплением → 3. Метод проб и ошибок.

3. **Верно ли утверждение?**  
   **Да.** Нейронные сети имитируют структуру мозга (нейроны, связи) для решения задач (распознавание, прогнозирование).

4. **Порядок этапов работы языковой модели:**  
   ```
   № | Этапы работы языковой модели  
   --|------------------------
   1 | Векторизация  
   2 | Обработка последовательности  
   3 | Оценка вероятностей  
   ```


5. **Верные утверждения о LLM:**  
   ```
   Знак + | Высказывание  
   -----|-------------
   +    | Основаны на архитектуре трансформеров  
        | Обучены на небольших объемах специализированной информации  
        | Используются только для анализа текстов  
   +    | Могут создавать и анализировать тексты  
   +    | Обучены на больших объемах разнообразной информации  
   ```

6. **Эссе: ML в беспилотных автомобилях (3–5 предложений)**  
   Машинное обучение позволяет беспилотным автомобилям воспринимать окружение и принимать решения. Компьютерное зрение (CNN) распознаёт дорожные знаки, пешеходов и другие машины. Алгоритмы прогнозирования предсказывают траектории движения объектов. Обучение с подкреплением помогает оптимизировать манёвры (перестроение, торможение). Данные с датчиков (лидары, камеры) обрабатываются в реальном времени, чтобы обеспечить безопасное движение.


7. **Почему важны вопросы «Что?», «Почему?», «Как?» в промпте?**  
   Эти вопросы структурируют запрос:  
   * **«Что?»** задаёт конкретную задачу (сгенерировать, объяснить, сравнить).  
   * **«Почему?»** уточняет контекст и цель (для школьника, для научной статьи).  
   * **«Как?»** определяет формат и ограничения (список из 5 пунктов, 100 слов).  
   Без них модель может дать слишком общий или нерелевантный ответ.


8. **Пример ML в электронной коммерции**  
   Персонализированные рекомендации: модель анализирует историю покупок, просмотры и поведение пользователя, чтобы предложить товары, которые он, скорее всего, купит. Это увеличивает конверсию и удовлетворённость клиентов.


9. **Схема взаимодействия человека с языковой моделью**  
   ```
   Пользователь → Промпт (текст) → Модель (векторзация → обработка → генерация) → Ответ (текст) → Пользователь
   ```
